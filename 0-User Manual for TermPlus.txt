
TermPlus 用户手册 - last edit on 2026.03.17 by xian wu

### 0、简介

本软件由 Xian Wu 个人开发，为单一 exe 文件的 GUI 程序，可在 windows 系统上运行（如果要在其他操作系统上运行需要使用源码重新打包），使用 python 3.10 和 VS2022 的环境，借助了 ChatGPT 的帮助 。

版本命名说明：由于 CrowdStrike 的原因，后续所有版本不再在文件名中包含打包日期，默认为 `TermPlus_2025.exe` 。

用户可以在打开后通过 **About** 菜单确认具体的版本打包日期（如 v260109 表示 2026 年 1 月 9 日打包） 。

开发该软件的主要目的为解决 Term 和 IPOP 软件的缺陷，满足对易用性需求（循环测试语句不好写，自动保存日志不方便，长期运行容易崩溃，循环发送会占用进程等） 。

该软件还增加了几个额外的功能，包括：
-对 Power Shell（CMD 终端）的支持，可以通过该 GUI 的脚本直接控制 power shell 。
-和 SCPI（程控电源）的支持（实现对指定 IP 的 linux 设备的文件写入操作和程控电源的掉电重启的需求） 。
-对 SSH 远程端口的访问，可以直接远程连接 SSH server 的虚拟串口 。
-可以通过简化后的 python 的代码获取串口回复的内容，进行匹配、判断和后续操作 。
-可以通过脚本开启/关闭 COM 口，以及修改 COM 口的配置。

该软件同时也增加了时间戳、上下键访问历史发送指令、简易循环脚本语法、脚本关键字高亮、自动清屏（长时间运行更稳定）和日志保存等功能 。


### 1、使用说明

1.1 打开串口

双击 `TermPlus_2025.exe`，窗口启动，选择串口下拉框，选择想要打开的 COM 口，选择合适的波特率（如果没有合适的波特率可以直接输入一个波特率），点击“打开串口” 。

如果启动程序时还没插入串口设备，等串口设备插入后，点击“检测可用串口”，则可以重新获取到可连接的串口设备 。

也可以点击“SSH 配置”进入 SSH 界面，可以远程访问其他设备的 SSH server 的虚拟串口，跟直接连接物理串口的效果是相同的 。

通过 "配置 SSH" 的按键弹出的窗口来设置 IP-账号-密码，例如 `10.166.1xx.xx - root - （密码为空）`，点击连接，等待连接成功或报告失败 。

串口或 SSH 虚拟串口连接成功后，可以看到对应的状态颜色指示块变成绿色 。


1.2 主窗口发送和接收

在左上角的主窗口中，直接输入命令，按下回车键则会发送出去，输入内容的末尾默认会增加一个“\n”字符，未输入任何内容时按下回车键，会单独发送一个“\n”字符。

接收到的内容，会直接显示在主窗口中，对于连续接收内容，每 0.1 秒或每收到 2048 个字符刷新显示一次 。

主窗口的输入支持 Windows 标准的 Ctrl+Z (撤销) 和 Ctrl+Y (恢复)。

为了不与撤销功能冲突，Linux 系统的快捷键映射为 Alt + c/d/z/l/a/e/k/u，分别对应 Linux 下 Ctrl 组合键产生的 ASCII 码 。

在主窗口和单行发送窗口中按上下键可以在最近发送过的命令中进行选择（仅对直接输入的内容生效，通过按键发送和脚本发送的内容不会被记录在这个历史记录中）。


1.3 单行发送和单行循环

在发送文本框中输入内容，点击“发送”按键，会将内容发送出去，如果勾选了“发送新行”，则会默认在内容的末尾增加一个“\n”，窗口启动后该框默认勾选。

点击“单行循环”按键，则会将发送文本框中的内容循环发送，间隔为循环间隔文本框中写入的时间，再次点击则会停止。

支持发送十六进制字符，发送 `\xA1` 会被替换为实际的十六进制数据“0xA1”发送 。

发送文本框也是一个下拉框，记录了历史发送命令。


1.4 自动保存

点击自动保存路径下方的按键，选择路径，在保存文件名文本框中写入想要的文件名，点击自动保存右侧的OFF按键，按键变成ON，则开启了自动保存功能。

该功能会创建一个.txt文件，文件名为指定的文件名+日期+时间，存放在指定的路径下。

点击ON，则按键切换为OFF，自动保存关闭。在手动关闭自动保存（点击切换为 OFF）时，软件会立即进行一次保存，然后结束对该文件的写入。

每10s会进行一次自动保存，如果保存文件的大小超过了指定的文件最大容量，则会新建一个文件继续保存。

每10s进行一次自动删除的判断，检测主窗口中字符数量，若超过 1M (100万)个字符，则自动删除主窗口中前半部分内容以防止内存占用过高导致卡顿（不影响自动保存的功能）。该功能默认勾选开启。

自动删除功能在脚本区域执行 `wait_for` 等函数时会暂时禁用，不影响实际使用 。


1.5 按键区域

点击“按键路径/选择”下方的按键，选择文件夹，下拉选择框会将该路径下的所有.ts文件显示出来，选择想要加载的文件，点击“加载”按键，则该文件中包含的内容都会加载到标签页区域。

单击按键标签页区域的按键，则该按键的多行字符串会逐行发送出去。

如果“等待符号”或”超时时间“为空，则默认每行发送间等待100ms。

如果“等待符号”中有单个字符或多个用逗号分隔开的字符，且“超时时间”中有数字，则每次发送后会在任意等待字符出现后再发送下一行，如果超时还没等到指定字符，则会停止发送并在主窗口中显示红色提示文字。

发送的命令中，如果有wait t或wait for x for t（还有其他一些特殊函数会执行特定动作，参见examples），则会执行延时和等待指定内容的延时操作，其他内容会直接被逐行发送出去。

右键点击标签页区域的按键，还可以编辑该按键的名称和内容，也可以右键-edit后，在弹出的窗口中进行：修改按钮显示名称、写入多行内容、发送当前行、发送当前行并移动到下一行（非空、非注释的行）、发送全部、保存、保存并退出等操作。

编辑窗口支持撤销/恢复、剪切、复制、粘贴等右键菜单。

编辑窗口中的多行内容，行首使用 `#` 的内容为注释内容，不会被发送，且在编辑窗口中以绿色斜体显示 。

按键标签页区域的内容，可以通过“保存”、“另存”按键进行保存和另存为。


1.6 脚本区域

脚本区域可编写 Python 代码并逐行执行。语法和缩进规则跟python一致。python自带库均可运行，有部分库函数已经导入该代码空间，参见后面的说明。

当点击“开始”按键后，会逐行运行脚本区域多行文本框内的每一行代码，对于send xxx和send_line(xxx,True,True)语句，“行间延时ms”文本框中有内容时，会使得send函数发送后延时对应的时间。

脚本区域的send x的下一行如果没有wait或wait for，则执行时会增加一条wait函数，wait的时间为默认间隔时间。

如果“循环次数”文本框中的数字为0，则会永久循环发送，如果为1，则发送1次后结束，如果为n，则循环n次后结束。

当前的循环次数会显示在“当前执行循环”处。

脚本运行过程中，可以使用暂停和停止按键，也可以在暂停后再次点击开始。

脚本区域的文本框中的内容，可以指定文件夹路径，选择下拉文本框中的.txt文件，点击打开，即会将该txt文件的所有内容加载到文本框中。

在文本框的内容修改后，也可以保存和另存为。

运行过程中，会逐行高亮正在运行的语句，wait和wait for x for t 语句会持续高亮直至等待结束或超时。

Python 关键字及自定义函数会自动蓝色高亮 。若运行出错，会弹出报错窗口显示错误行号、变量信息和主窗口文本，并记录在 `script_error_info.txt` 中 。

运行自定义脚本期间会临时关闭主窗口的撤销功能，以防内存无限增长 。



1.7 界面调整按钮

在左上方有“按键”、“脚本”、“配置”、“主题”等按钮，前三个按键按下后会切换对应区域的显示和隐藏，主题按键按下后会切换浅色和深色界面。

窗口的标题栏和菜单栏无法进行颜色修改。


### 2、细节说明

2.1 主窗口交互

串口接收到的内容，会直接显示在主窗口中，如果接收内容的末尾是回车符或换行符，则会立即显示。

如果接收到的内容没有以回车符或换行符结尾，则会等待0.01秒后才将接收到的内容作为一次接收显示到主窗口中。

在主窗口中输入的字符，会暂存在buffer中，按下enter键时才会发送并清空buffer。

主窗口支持撤销和恢复操作，可以使用ctrl+z和ctrl+y。

这与linux的停止按键冲突了，因此将linux的快捷键更改为了Alt+c/d/z/l/a/e/k/u（linux快捷键实际上是对对端硬件设备发送了特定的二进制数据，对于未实现的快捷键也可以查询到二进制字符串后单独发送）。

在主窗口中无法使用键盘的方向键进行自由的移动，仅能在输入的字符串区域内进行左右移动，上下键默认为历史命令的浏览键。

串口空闲检测时间优化为0.001s，确保连续大流量传输时的稳定性。

各类文本编辑窗口（主窗口、脚本框等）的撤销步数限制为20 步。

主窗口对 Backspace 和方向键进行了限制，使其仅能在输入的字符串区域内移动和操作，避免破坏已接收的历史数据 。



2.2 自动保存与加载

运行exe时，会检测是否存在.ini文件及其配置，如果存在则会将相关配置加载到整个界面。

关闭程序时，会将当前配置及历史命令记录保存到 `.ini` 文件中，下次打开自动恢复（自动恢复的功能对于脚本区域的路径存在bug，在20260213版本修复）。

2.3 脚本自定义函数列表

该文本框中的内容，相当于一个可供python代码运行的空间，点击“开始”，会将所有的行合并成一个可执行的代码字符串，然后执行代码。

针对send x，wait t， wait for x for t三个函数，进行了特殊处理，这三个函数不满足python语法，但会转换成符合python语法的函数去执行。

send_line(text,addNewline=True,insertDisplay=True)
wait(seconds)
wait_for(target_string, over_time)

可以参考Example Scripts文件夹中的txt文档，了解for while循环，以及所有自定义函数的使用方法。

-增加了对Power Shell和SCPI函数的支持，参见对应的示例。
-增加了获取主窗口文本的函数，参见示例。
-增加了显示show_str(“xxx”)函数。

目前所有支持的自定义函数包括：
	send xxx,等价于send_line(xxx,True,True)
	wait x，等价于wait(x)
	wait for xxxx for t，等价于wait_for(xxx,t)
	wait_for_any(["xxx","yyy","zzz","www"],t)
	messagebox.showinfo("title_str", "content_str")
	raise StopLoopException("stop")
	show_str(str_to_show)
	show_date_time()
	get_last_n_lines(line_amount)
	get_future_n_lines_or_till_overtime(line_amount,overtime)
	get_future_lines_till_str_or_overtime(str_to_end,overtime)
	connect_serial()
	close_serial()
	toggle_serial()
	set_port("COMx")
	set_baudrate("xxxx")
	visa_init()
	visa_query()
	visa_write()
	visa_read() read函数未进行展示，电源的开关不需要进行单独的read操作
	open power shell
	power shell send xxx

除了自定义函数，脚本区域可以运行大部分python内置的代码，如果有不支持的代码，也可以尝试使用termplus的源代码增加包含的库之后进行打包，需要同时在代码的“local_namespcae”包含需要使用的函数名。

脚本区域如果检测到代码有问题或运行时出现异常，会停止运行并报告该异常，可以从弹出的窗口中观察到具体的报错内容。

该报错会生成一个script_error_info.txt，可以在exe相同目录下看到其内容。

2.4 加载脚本

可以加载后缀为.ts的文件，加载时会依次尝试 **GBK, UTF-8, UTF-8-sig, Latin-1** 解码，并记住解析成功的编码方式以便后续保存，防止乱码 。使用Notepad++可以查看或修改其编码，应当为ANSI或Latin-1编码，否则可能无法加载或出现乱码。

2.5 给窗口命名

在右上角的窗口名下方的文本框中输入文字，窗口的标题会随着一起变化。

2.6 Power Shell （CMD）

增加了对Power Shell的支持，Power Shell是CMD的完整版，可以实现SCP等操作，参见Example Scripts。

2.7 SCPI

增加了对SCPI的支持，可以控制程控电源等设备，参见Example Scripts。

2.10 时间戳

勾选时间戳box之后，发送和接收的内容前都会增加一个“Send at xx:xx:xx.xxx”和“Receive atxx:xx:xx.xxx”，分别为蓝色和绿色。

接收时间戳会在第一个字符接收到后显示，显示后如果超过0.1秒没有持续接收到字符，才会显示下一个接收时间戳。

2.11 捐赠

“About”菜单中增加了捐赠选项，可扫描二维码支持作者。
